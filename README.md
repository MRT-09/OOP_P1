# OOP Project 1

## Банкова система

Проектът представлява обектно-ориентирана банкова система, която позволява управление на потребители, различни видове банкови сметки, транзакции и банкови карти. Системата симулира реални банкови операции като депозити, тегления, преводи и генериране на отчети.

## Функционалности

### Създаване на потребител

Позволява добавяне на нов потребител със собствен уникален идентификатор и списък от банкови сметки.

### Създаване на банкова сметка

Потребителят може да създава различни типове сметки (разплащателна, спестовна, кредитна).

### Депозиране на средства

Добавяне на пари към дадена сметка, като балансът се увеличава.

### Теглене на средства

Изваждане на пари от сметка с проверка за достатъчна наличност или кредитен лимит.

### Превод между сметки

Прехвърляне на средства между две сметки (в рамките на системата).

### История на транзакциите

Съхраняване и преглед на всички извършени операции по дадена сметка.

### Изчисляване на лихва (за спестовни сметки)

Автоматично начисляване на лихва върху наличния баланс.

### Кредитен лимит (за кредитни сметки)

Позволява балансът да стане отрицателен до определен лимит.

### Генериране на месечен отчет

Обобщение на всички транзакции и текущото състояние на сметката.

### Засичане на подозрителни транзакции

Маркиране на транзакции над определена сума (напр. 10 000) като потенциално рискови.

## Клас диаграма (текстово описание)

### Основни класове

#### User

private:

- id : int
- name : string
- accounts : vector\<Account*\>

public:

- addAccount(Account*)
- getAccounts()

### Account (абстрактен клас)

protected:

- accountNumber : string
- balance : double

public:

- virtual deposit(double)
- virtual withdraw(double) = 0
- virtual transfer(Account*, double)
- virtual printInfo()

#### CheckingAccount : public Account

public:

- override withdraw(double)

#### SavingsAccount : public Account

private:

- interestRate : double

public:

- applyInterest()
- override withdraw(double)

#### CreditAccount : public Account

private:

- creditLimit : double

public:

- override withdraw(double)

#### Transaction

private:

- amount : double
- date : string
- type : string

#### Card

private:

- cardNumber : string
- linkedAccount : Account*
- limit : double

#### Bank

private:

- users : vector\<User\>
- transactions : vector\<Transaction\>

public:

- addUser(User)
- processTransaction(...)
- generateReport()

### Връзки между класовете

User има множество Account (композиция).

Account е базов клас за: CheckingAccount, SavingsAccount, CreditAccount

Account има Transaction

Card е свързана с Account

Bank управлява всички обекти.

## Обосновка на дизайн решения

1. Използване на абстрактен клас Account Класът Account е направен абстрактен, защото няма смисъл да съществува самостоятелно — той служи като обща основа за различни типове сметки. Това позволява използване на полиморфизъм и избягва дублиране на код.

2. Наследяване на различни типове сметки SavingsAccount, CheckingAccount и CreditAccount наследяват Account, защото всички те споделят общи характеристики (баланс, номер), но имат различно поведение при теглене и допълнителни функционалности (лихва, кредитен лимит).

3. Капсулация на полето balance Полето balance е направено protected/private, за да се предотврати директна промяна отвън. Всички операции минават през методи като deposit() и withdraw(), което гарантира контрол и валидност на данните.

4. Използване на композиция между User и Account Потребителят притежава сметки, което е естествена връзка „has-a“. Ако потребителят бъде изтрит, неговите сметки също губят смисъл в системата.

5. Централен клас Bank Класът Bank управлява всички потребители и транзакции, което централизира логиката и улеснява контрола върху системата.
